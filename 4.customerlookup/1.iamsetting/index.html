IAM 설정

Lambda 에서 Connect 와 DynamoDB 에 접근하기 위해서는 권한이 필요합니다. IAM 서비스에서 권한을 설정할 수 있습니다. 권한은 Role을 생성하면서 정의합니다. Role은 필요한 권한의 종류를 정의할 수 있습니다. 더 자세한 정보는 다음 URL을 참고하시기 바랍니다. IAM Role : https://docs.aws.amazon.com/IAM/latest/UserGuide/id_roles.html

  1. AWS Console https://console.aws.amazon.com 로 접속합니다.

Connect Console 이 아닌 AWS Console 로 접속합니다.

  1. 상단의 검색창에서 IAM 서비스를 검색하여 이동합니다.

  2. 왼쪽 메뉴에서 Roles를 선택합니다.

  3. Create role 버튼을 클릭합니다.

  4. 권한을 가지는 주체를 선택하는 화면입니다. 본 실습에서는 Lambda를 이용하여 DynamoDB의 데이터를 조회하고 Connect 에 전달할 것이기 때문에, 주체로 Lambda 를 선택하고 Next: Permissions 버튼을 클릭하여 진행합니다.

  5. 세부 권한을 설정합니다. 아래 3가지의 Role을 검색하여 체크박스에 체크합니다. 모두 체크했으면 Next: Tags 버튼을 클릭하여 진행합니다.

    • AWSLambdaBasicExecutionRole
    • AmazonDynamoDBFullAccess
    • AmazonConnect_FullAccess

  6. Next: Review 버튼을 클릭하여 다음으로 진행합니다.

  7. Role 이름을 설정합니다. 아래 Text를 복사해서 정확히 Role 이름을 입력합니다. 이후 다시 사용해야하기 때문에 Role 이름을 기록해둡니다. 아래 Policies 항목에 3가지 Permission 이 모두 제대로 설정되었는지 확인합니다. 만약 제대로 설정이 되어있지 않다면 Previous 버튼을 클릭하여 6번 항목으로 다시 이동해서 재설정합니다. 모두 확인이 되었으면 Create Role 버튼을 클릭하여 Role 설정을 완료합니다.

    role-connect-imd
