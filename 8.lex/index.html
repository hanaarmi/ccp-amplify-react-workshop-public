8. 챗봇

이번 장에서는 챗봇(보이스봇)을 구성하여 고객으로부터 DTMF 신호를 받는 대신, 음성으로 입력을 받아 원하는 Queue (대기열)로 이동할 수 있도록 구성합니다. Amazon Lex 는 Managed Chatbot 서비스로써, 손쉽게 챗봇 또는 보이스봇을 구성할 수 있는 서비스입니다. 자세한 내용은 다음 링크를 참고하세요. Amazon Lex:https://aws.amazon.com/ko/lex/

Amazon Lex 설정 (Chatbot 설정)

  1. AWS 콘솔에 접속합니다. Lex 서비스로 이동합니다.

  2. Get Started 버튼을 클릭하여 계속 진행합니다.

  3. Create bot 버튼을 클릭합니다.

  4. 아래 화면과 같이 설정합니다. Bot name 은 아래 Text를 복사해서 정확하게 입력합니다. 다 설정했으면 Next 버튼을 클릭하여 진행합니다.

    bot-connect-imd

  5. 아래와 같이 설정한 후 Done 버튼을 클릭하여 진행합니다.

  6. 초기 설정을 완료하고 나면 아래와 같이 Intent 를 입력할 수 있습니다. 아래와 같이 Intent name 을 sales 로 입력합니다.

  7. 아래로 스크롤하여 Sample utterances 를 입력합니다. 이후 화면 아래 Save intent 버튼을 클릭하여 설정을 저장합니다.

챗봇의 역할은 Sample utterances (발화 예시) 를 기반으로, Machine Learning 을 통해 유사한 발화 내용을 인식하여, 알맞은 Intent (의도)를 제시하는 것 입니다. 여기에서 입력한 Sample utterances 를 기반으로 Amazon Lex ML 엔진이 이와 비슷한 유형의 발화를, 지정한 Intent 로 인식할 수 있도록 ML 모델을 빌드할 수 있습니다.

  1. sales Intent 입력을 완료했으면, 아래와 같이 리스트로 돌아갑니다.

  2. Add intent 밑의 Add empty intent 버튼을 클릭하여 새로운 Intent 를 입력합니다.

  3. inquiry Intent 를 추가합니다.

  4. inquiry Intent 에 대해 아래와 같이 Sample utterances 를 입력합니다. 이후 Save intent 버튼을 클릭하여 저장합니다.

  5. support Intent 에 대해서도 마찬가지로 8번 부터 반복하여 아래와 같이 Sample utterances 를 입력하고 저장합니다.

sales, inquiry, support 3개의 intent 를 모두 소문자로 정확히 입력합니다.

  1. 이제 아래 Build 버튼을 클릭하여 Machine Learning Model 을 빌드합니다.

  2. Build 가 완료되면 아래와 같이 상단에 메시지가 출력됩니다. 이제 Model 이 잘 빌드되었는지 Test 버튼을 클릭하여 테스트해봅니다.

  3. 아래와 같이 유사한 문장을 입력했을 때 Intent 가 잘 인식되는 것을 확인할 수 있습니다. 만약 의도한대로 Intent 가 분석되지 않는다면, Sample utterances에 데이터를 더 입력하고 다시 Build 하여 정확도를 더 높일 수 있습니다. 또한 Build 한 Version 별로 별도로 관리하고, Connect 에 적용할 때에도 Version 을 구분하여 적용할 수 있습니다.

Contact Flow 챗봇 적용

  1. AWS 콘솔로 접속합니다. Connect 서비스로 이동한 이후 인스턴스를 클릭하여 이동합니다.

  2. 아래와 같이 위에서 생성한 챗봇을 등록합니다.

  3. Connect 콘솔로 접속합니다. Contact flow 메뉴로 이동하여 ConnectIMD5 Contact Flow 를 클릭하여 편집 화면으로 진입한 이후, Save as 로 아래와 같은 이름으로 다시 저장합니다.

본 실습에서는 기존 ConnectIMD5 는 DTMF 입력을 받는 시나리오, ConnectIMD5.1은 Lex 로 보이스봇 시나리오를 실행하도록 구분하고 있습니다.

  1. Get customer input Command block 을 클릭하여 아래 설정과 같이 편집합니다. Save 버튼을 클릭하여 설정을 저장합니다.

  2. Get customer input Command block 의 sales, inquiry, support 분기와 Set working queue Command block 들을 다시 연결해줍니다. 이후 Save 를 하고 Publish 까지 완료합니다.

  3. ConnectIMD4 를 편집합니다.

  4. 아래 화면과 같이 Transfer to flow Command block 을 ConnectIMD5.1 로 설정을 변경해줍니다. Save 이후 Publish 까지 완료합니다.

다시 DTMF 를 이용한 시나리오로 돌아가려면, 위 7번에서 ConnectIMD5 로 설정하도록 합니다.

챗봇을 활용한 시나리오를 구성해봤습니다. 다음장 부터는 Callback, Case playback, Last Agent Routing 시나리오에 대해 실습합니다.
