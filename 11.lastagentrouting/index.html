11. Last Agent Routing 시나리오

Connect에서는 유연한 Contact flow 디자인으로 여러가지 Routing 을 구성해볼 수 있습니다. 이번 장에서는 Last Agent Routing 을 구성해보겠습니다. 이전 4장에서 DynamoDB의 고객 정보 항목 중 lastagent 항목에 저장되어 있는 직전 상담원 기록을 이용해, 현재 그 상담원이 상담을 받을 수 있는 상태(Available)이면 연결하는 시나리오입니다. 그리고 또한, 상담원과 통화가 성공하면 DynamoDB의 현재 상담주인 상담사로 lastagent 항목을 업데이트하는 실습도 해보겠습니다.

Last agent Update Lambda 함수 생성

  1. AWS 콘솔로 접속하여 Lambda 서비스로 이동합니다.

  2. Create function 버튼을 클릭하여 Lambda 함수를 생성합니다.

  3. 아래와 같이 설정합니다. 함수 이름은 아래 Text를 복사하여 정확히 입력합니다.

    func-connect-lastagent

 #Import libraries and set table name
 import json
 import boto3
 import os

 tablename= os.environ['tablename']
 dynamodb = boto3.resource('dynamodb')

 def lambda_handler(event, context):
 #Customer phone number and last agent deatils passed from Amazon Connect via JSON
 phonenumber = event ['Details']['ContactData']['CustomerEndpoint']['Address']
 lastagent = event['Details']['ContactData']['Attributes']['lastagent']

 #customer lookup to DynamoDB via Customer phone number
 table = dynamodb.Table(tablename)
 response = table.get_item(Key={'phonenumber' : phonenumber})

 #If record exists update the last agent field
 if 'Item' in response:
     table.update_item(
     Key={'phonenumber': phonenumber},
     UpdateExpression="SET lastagent = :var",
     ExpressionAttributeValues={':var': lastagent}
     )
     return {'message': 'updated'}

     #If no record exists create a new record with the phone number and last agent field
 else:
     table.put_item(
     Item={
         "phonenumber": "" + phonenumber + "",
         "lastagent": "" + lastagent + "",
         }
     )
     return { 'message': 'added'}

위 코드는 Production 용도로 사용하는 코드가 아닌 실습용으로 사용하는 코드입니다. Production 에 반영하기 전 꼭 리뷰를 수행하시기 바랍니다.

  1. Configuration 탭으로 이동한 후 Environment variables 메뉴에서 tablename 을 아래와 같이 text를 복사하여 정확히 입력합니다.

     table-connect-imd

  2. 다시 Code 탭으로 이동한 후 Deploy 버튼을 클릭하여 함수를 배포합니다.

  3. 아래 화면과 같이 배포가 완료됐는지 확인합니다.

  4. AWS 콘솔로 이동하여 Amazon Connect 서비스로 이동합니다. (Connect 콘솔이 아닙니다.)

  5. 이전에 생성한 인스턴스를 클릭하여 진행합니다.

  6. 아래와 같이 Contact flows 메뉴에서 AWS Lambda 항목의 설정을 진행합니다. 기존에 등록했던 func-connect-imd 함수 외에 func-connect-lastagent 함수까지 등록을 합니다. 아래 화면과 같이 총 2개의 함수가 등록이 되어 있어야합니다. (꼭 +Add Lambda Function 버튼을 클릭하여 저장하여야 합니다.)

Customer whisper flow 에서 lastagent 정보 업데이트 하기

  1. 이제 Connect 콘솔로 접속하여 Default customer whisper 를 클릭하여 편집합니다.

Customer whisper flow 는 고객이 상담원과 통화에 연결되었을 때, 고객의 시나리오를 지정할 수 있습니다. Connect 에 초기부터 저장되어 있는 Default customer whisper flow 는 기본적으로 통화 연결 시 beep 음을 내도록 구성되어 있으나, 위와 같이 변경할 수 있습니다. 또는 별도의 Customer whisper flow를 구성하여 Set whisper flow Command block으로 전역 설정할 수 있습니다.

  1. 아래 시나리오를 작성합니다. 각각의 Command Block 설정은 아래 설명을 참고하여 작성합니다.

  2. Set contact attributes Command block 을 왼쪽 리스트 중 Set 메뉴에서 Drag&Drop 으로 편집 화면으로 가져옵니다. 아래와 같이 설정한 후 저장합니다. lastagent Attribute 에 현재 연결된 상담원의 Username 정보를 저장합니다. 이 정보는 다음에 호출할 lambda 함수로 보내집니다.

  3. Invoke AWS Lambda function Command Block 을 왼쪽 리스트 중 Integrate 메뉴에서 Drag&Drop 으로 편집 화면으로 가져옵니다. 이전에 생성한 Lambda 함수 func-connect-lastagent 를 선택합니다. Save 버튼을 클릭하여 설정을 저장합니다. Lambda 함수를 호출하여 직전 상담원 정보를 DynamoDB 테이블에 반영합니다.

  4. 모든 시나리오를 작성했으면, Save 하고 Publish 를 합니다.

Last Agent Routing 시나리오 작성을 완료했습니다. 테스트를 통해 직전 상담원 연결이 잘 수행되는지 확인해보시기 바랍니다. 위 Get customer input 에서 상담원 연결을 선택하지 않는 분기를 선택한 후 Customer whisper flow에서 상담원 정보가 DynamoDB에 올바르게 업데이트 되는지 테스트해봅니다. 이후 다시 통화 테스트로 Get customer input 에서 직전 상담원 연결로 분기하여 해당 상담원에게 잘 연결이 되는지 확인합니다. Cloudwatch를 사용하며 명시적으로 확인할 수 있습니다.
