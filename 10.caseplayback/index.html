10. Case Playback 시나리오

지금까지 Lex 를 활용한 Machine Learning 기반의 콜센터를 구축해봤습니다. 이제부터는 조금 더 응용된 시나리오를 구성해보겠습니다. IVR 시나리오를 Contact flow 에서 직접 작성해보고 잘 작동하는지 확인합니다. 이번 장에서는 Case Playback 시나리오를 구성해보겠습니다. SSML 을 이용하여 Polly 에서 동적인 메시지를 멘트로 들려줄 수 있도록 합니다.

  1. Connect 콘솔로 접속합니다. Contact flow 메뉴에서 ConnectIMD7 를 클릭하여 편집합니다.

  2. 아래 시나리오를 작성합니다. 각각의 Command Block 설정은 아래 설명을 참고하여 작성합니다.

  3. Check contact attributes Command Block 을 왼쪽 리스트 중 Branch 메뉴에서 Drag&Drop 으로 편집 화면으로 가져옵니다. 아래와 같이 설정한 후 Save 버튼을 클릭하여 설정을 저장합니다. message 변수를 확인하여 Lambda 함수로 고객 정보를 가져오는 것을 성공했는지 확인합니다.

  4. Get customer input Command Block 을 왼쪽 리스트 중 Interact 메뉴에서 Drag&Drop 으로 편집 화면으로 가져옵니다. 아래 Text를 복사해서 입력합니다. 이후 나머지는 아래와 같이 설정한 이후 Save 버튼을 클릭하여 설정을 저장합니다. SSML 을 이용하여 동적인 변수를 Polly 를 통해 읽어줄 수 있도록 설정합니다.

    <speak>
$.Attributes.firstname the latest case you have open is <say-as interpret-as="digits"> $.Attributes.casenumber </say-as>
Press 1 to hear the latest case comments.
Press 2 to speak to an agent about a new case.
</speak>

  5. Play prompt Command Block 을 왼쪽 리스트 중 Interact 메뉴에서 Drag&Drop 으로 편집 화면으로 가져옵니다. 아래 Text 를 복사해서 입력합니다. Save 버튼을 클릭하여 설정을 저장합니다.

    <speak>
Case Number
<break time="1s"/>
<say-as interpret-as="digits">$.Attriburtes.casenumber</say-as>
<break time="1s"/>
$.Attributes.casedetail
</speak>

  6. Get customer input Command Block 을 왼쪽 리스트 중 Interact 메뉴에서 Drag&Drop 으로 편집 화면으로 가져옵니다. 아래 Text 를 복사해서 입력합니다.아래와 같이 설정한 이후 Save 버튼을 클릭하여 설정을 저장합니다.

    Press 1 to speak to an agent

  7. Transfer to flow Command Block 을 왼쪽 리스트 중 Terminate / Transfer 메뉴에서 Drag&Drop 으로 편집 화면으로 가져옵니다. ConnectIMD8 로 설정합니다. Save 버튼을 클릭하여 설정을 저장합니다.

  8. 모든 시나리오를 작성했으면, Save 하고 Publish 를 합니다.

Amazon Polly 에서 지원하는 SSML Tag는 다음 링크를 참고하세요. https://docs.aws.amazon.com/ko_kr/polly/latest/dg/supportedtags.html

Case playback 과 관련된 시나리오를 모두 작성하였습니다.
